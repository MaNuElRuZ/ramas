He creado dos carpetas: capeta casa y carpeta instituto. En la carpeta casa he clonado un documento que tenía varios commits para poder trabajar con ella. He hecho un git checkout -b para poder crear una nueva rama llamada "nuevarama". Una vez creada he editado el documento, lo he añadido y le he puesto un nombre al commit. como es en la "nuevarama" no aparecerá los cambios hechos en la rama master.
Una vez hecho todo esto comprobaremos las ramas con git branch. Apareceran las dos ramas, pudiendose cambiar con el comando git checkout y el nombre de la rama.
Le haremos un git pull a la carpeta instituto para poder bajar todas las ramas a su ultima versión. Una vez comprobado todo lo que hemos hecho podremos hacer el git merge que sirve para fusionar las ramas. Esto lo haremos en donde queremos que se vea reflejado la fusión acompañado con el nombre de la rama. Despues al hacer el git log en la rama que hayamos querido hacerla veremos como se mezclara los commits de una y otra rama.
